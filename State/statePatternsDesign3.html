Design #3: Pushing Information

In this third design, when this ball is asked whether it intersects with some other ball, it sends its information to the other ball, and asks that other ball to compute the intersection. 

      // A Ball3 is an object of any class that implements Ball3.
      
      interface Ball3 {
      
          // Does the given ball intersect with this one?
      
          boolean intersects (Ball3 b);
      
          // GIVEN: the x, y, r for some ball
          // RETURNS: does that ball intersect this ball?
      
          boolean intersectResponder (int x, int y, int r);
      }
