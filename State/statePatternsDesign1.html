Design #1: Balls as Data Structures

Design #1: Just use objects in place of structs. We equip our objects with methods that get each field, and do our computation outside of any object.

Here is the interface and a sample class definition in this style: 

      // A Ball1 is an object of any class that implements Ball1.
      
      interface Ball1 {
          // RETURNS: x or y coordinates or radius of this ball,
          //     in pixels
      
          int getX ();
          int getY ();
          int getR ();
      }
      
      class Ball1class implements Ball1 {
          int x;    // x coordinate of this ball
          int y;    // y coordinate of this ball
          int r;    // radius of this ball, in pixels
      
          (// Java constructor and interpretations omitted)
      
          public int getX () { return x; }
          public int getY () { return y; }
          public int getR () { return r; }
      }
