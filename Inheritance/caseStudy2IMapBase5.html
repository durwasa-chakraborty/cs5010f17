Abstract Methods of the Abstract Base Class

The base class declares one public abstract method, extend<K,V>, which was listed in the IMap<K,V> interface. That method must be public because it was listed in an interface.

The base class declares three more abstract help methods. Those methods should not be public, because they should not be called outside of the base class and its subclasses. To make it possible for subclasses to be defined in a different package, we need to reduce the access protection of those help methods by declaring them protected. 

      abstract class IMapBase<K,V> extends AbstractMap<K,V> implements IMap<K,V> {
      
          ...
      
          // public methods listed by IMap<K,V> interface
      
          // Returns an IMap like this one except the given key is associated
          // with the given value.  The extend operation does not modify this
          // IMap.
      
          public abstract IMap<K,V> extend (K key, V value);
      
          ...
      
          // abstract help methods
      
          // Returns true iff this map has a mapping for the given key.
      
          protected abstract boolean hasKey (K key);
      
          // Returns the value associated with the given key.
      
          protected abstract V getVal (K key);
      
          // Returns an iterator for the keys of this IMap.
      
          protected abstract Iterator<K> keyIterator();
      }
