Packages in Java

The misleading name of the protected access modifier gives you some idea of why we have been defining all classes in the default package. The Java package system is moderately confusing, especially for programmers who are just learning Java.

We can't avoid packages completely, however, because we need to import parts of packages that are part of Java's standard libraries.

For the IMapBase<K,V> class, these are the things we need to import from the java.util package: 

      import java.util.Map;
      import java.util.Map.Entry;
      import java.util.AbstractMap;
      import java.util.AbstractMap.SimpleImmutableEntry;
      
      import java.util.Set;
      import java.util.AbstractSet;
      
      import java.util.List;
      import java.util.ArrayList;
      
      import java.util.Iterator;

Java's import declarations are analogous to the import declarations of Scheme or the require declarations of Racket. Java packages are analogous to Scheme libraries and Racket modules.

As in Scheme, an import declaration can import named parts of a package or it can import all parts exported by a package. The import declarations shown above import one class per declaration. To import everything that can be imported from the java.util package, a Java programmer would write: 

      import java.util.*;
