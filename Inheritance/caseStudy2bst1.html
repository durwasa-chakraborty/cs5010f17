Binary Search Trees

Finally, we are ready to implement binary search trees.

We will use separate classes to represent empty and non-empty trees. To reduce duplication of code, both classes will inherit from the same abstract base class.

The purpose statement (data definition) for that base class appears below. Its two constructor templates refer to a static factory method for creating empty trees, and to a dynamic factory method for creating non-empty trees. 

      // Constructor templates for IMapBST<K,V>:
      //
      //     IMapBST.empty(c)
      // Interpretation:
      //     c is the comparator to be used by the map
      //
      //     this.make (k, v, left, right)
      // Interpretation:
      //     k and v are the key and value to be used at the root
      //     left is the left subtree
      //     right is the right subtree
      // Where:
      //     k is distinct from all keys in the left and right subtrees
      //     k is greater than all keys in the left subtree
      //     k is less than all keys in the right subtree
      //     the left and right subtrees obey those invariants, recursively
