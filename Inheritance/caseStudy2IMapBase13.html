The toString Method

Although the toString method does not have to be compatible with equals, it is a good idea to arrange for equal objects to print the same.

That constrains the information that can be revealed by the toString method. Here, for example, the toString method shouldn't reveal the keys of an unsorted IMap, because there is no canonical order in which those keys could be listed. 

          public String toString () {
              System.out.println (super.toString());
              return "an IMap of size " + size() + " and hash " + hashCode();
          }

That completes our definition of the IMapBase<K,V> abstract base class.

Defining that abstract base class was the hard part. We can now redefine the IMapList<K,V> class so it inherits from the abstract base class, and then proceed to define a IMapBST<K,V> class that inherits from the abstract base class and implements a SortedIMap<K,V> of binary search trees.
