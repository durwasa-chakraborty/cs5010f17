Abstract Base Class for SortedIMap<K,V>

We don't have to override the hashCode() method, because the inherited hashCode() method still returns the same result for any maps that are equal according to our new definition of the equals method, but the new definition of equals allows us to write a hashCode() method that depends on the order in which keys are generated. That allows us to write a better hashCode() method.

This hashCode() method below doesn't take advantage of that, but it does emphasize the fact that computing a hash code for sorted maps can be done differently than for unsorted maps. 

          public int hashCode () {
              Iterator<K> it = keyIterator();
              int result = 2005052521;
              while (it.hasNext()) {
                  K key = it.next();
                  V val = getVal (key);
                  int keyHash = key.hashCode();
                  int valHash = val.hashCode();
                  result = result - 978278598 * keyHash + 1048370044 * valHash;
              }
              return result;
          }

We can override the toString() method as well. We could, for example, define the toString() method so it shows every tenth key up to some maximum number of keys. Doing so is left as an exercise for the reader.

That completes our definition of the SortedIMapBase<K,V> class.
