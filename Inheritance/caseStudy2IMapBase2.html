The IMapBase<K,V> Class

The purpose statement for the IMapBase<K,V> class tells programmers what they have to do when defining subclasses that inherit from it: 

      // Abstract base class for classes that implement IMap<K,V> or SortedIMap<K,V>.
      //
      // To define a class that implements IMap<K,V>:
      //     make that class a subclass of this base class
      //     within that class, define the following abstract methods:
      //         public    IMap<K,V>   extend(K key, V value)
      //         protected boolean     hasKey(K key)
      //         protected V           getVal(K key)
      //         protected Iterator<K> keyIterator()
      //     within that class, override the following methods:
      //         public    int         size()
      //
      // To define a class that implements SortedIMap<K,V>:
      //     make that class a subclass of SortedIMapBase<K,V>
      //     define or override the methods listed above
      //     define comparator()

The public methods are listed by one or more interfaces, so they have to be public.

The protected methods are abstract help methods that shouldn't be called outside of the base class and its subclasses. That's why they aren't public.

Access Modifiers

In Java, protected is an access modifier with the remarkable property that declaring a method to be protected makes it less protected than if no access modifier had been written at all.

If no access modifier had been written at all, the methods would not be accessible outside of the package in which the base class is defined.

We want those methods to be accessible in all subclasses of the base class. To accomplish that, we need to make them less protected. Declaring them to be protected makes them less protected: they become accessible in all subclasses, even in subclasses defined outside of the package in which the base class is defined.
