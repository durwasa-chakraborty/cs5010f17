Tail Calls in Java

Example: In 

      return this.operator().value(env).asFunction().body().value(newenv);

the entire expression starting with this and ending with the right parenthesis immediately before the semicolon is in tail context. That expression is a call to the value method for a lambda expression, passing newenv as argument, with the result of the call to body() as the receiver.

That one method call is a tail call. The following calls are not tail calls, because they do not occur in a tail context: 

      this.operator()
      this.operator().value(env)
      this.operator().value(env).asFunction()
      this.operator().value(env).asFunction().body()

For each of those calls, there is more work for this method to do before it can return to its own caller. For the tail call, there is no more real work to do. All this method would do with the result of the tail call is return it to its caller.

Why should this method have to act as middleman? It could just tell the method it's calling to return directly to the ultimate consumer.
