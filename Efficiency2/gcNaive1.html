Naive Benchmark

It's easy to write naive micro-benchmarks that compare the performance of explicit deallocation in C or C++ against garbage collection in Java, Scheme, or similar programming languages.

We might, for example, write a benchmark that allocates a million linked lists, each of length 1000, totalling one billion calls to new or malloc, and an equal number of calls to free or delete.

On my usual benchmark machine, taking the median of three runs, the results were:

System Time in Seconds (not including compilation)
Java 2.898
C++ (gcc) 20.412

Java is 7 times as fast as C++ on this naive micro-benchmark. Why? Because this naive micro-benchmark is very nearly a best case for generational garbage collection.
