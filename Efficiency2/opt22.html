Optimizations that Improve Asymptotic Efficiency

Common subexpression elimination avoids computing the value of an expression if the value of that expression has already been computed and can easily be remembered within a register or stack slot.

Common subexpression elimination is a fairly straightforward optimization that will be performed by all optimizing compilers worthy of the name. Even so, it can occasionally improve the asymptotic running time of a program.

In a purely functional language, two calls to the same function with the same arguments will always return the same value, and those calls will never have any side effects. Common subexpression elimination can therefore convert a function such as 

          # Returns a tree of height n.
      
          makeTree(n)
              if n = 0
                then emptyTree
                else makeNode (makeTree (n - 1),
                               makeTree (n - 1))

into 

          # Returns a tree of height n.
      
          makeTree(n)
              if n = 0
                then emptyTree
                else let tmp1 = makeTree (n - 1)
                     in makeNode (tmp1, tmp1)

That reduces the running time from Ω(2n) to O(n).
